每个epic包括：
- 扩展的目标和价值主张
- 完整的story分解和用户故事
- 每个story的验收标准
- 故事排序和依赖关系

**Epic排序原则：**

- Epic 1建立粤语语音交互的基础设施和初始功能
- 后续epic渐进式构建，每个都交付重要的端到端价值
- Epic内的故事垂直切片并按顺序排列
- 无前向依赖 - 每个故事只基于之前的工作构建

## 🎙️ Epic 1: 语音交互核心系统 (Stories 1.1-1.15)

**目标**: 建立高准确率的粤语语音交互核心能力
**价值主张**: 产品核心竞争力，直接决定用户体验
**预估工期**: 8周
**Story总数**: 15个

### Story 1.1: 语音唤醒和基础识别

**As a** 家庭用户,
**I want** 能够通过语音唤醒机器人并让机器人识别我的基本指令,
**So that** 我可以开始与机器人进行自然的语音交互。

**验收标准:**
1. 支持自定义唤醒词，默认"小X"
2. 唤醒词识别准确率 >95% (安静环境下)
3. 唤醒响应时间 <1秒
4. 基础指令识别准确率 >90%
5. 支持普通话和粤语混合识别

**Prerequisites:** 无

## 📖 Story Guidelines Reference

### Story Format

```
**Story [EPIC.N]: [Story Title]**

As a [user type],
I want [goal/desire],
So that [benefit/value].

**Acceptance Criteria:**
1. [Specific testable criterion]
2. [Another specific criterion]
3. [etc.]

**Prerequisites:** [Dependencies on previous stories, if any]
```

### Story Requirements

- **Vertical slices** - Complete, testable functionality delivery
- **Sequential ordering** - Logical progression within epic
- **No forward dependencies** - Only depend on previous work
- **AI-agent sized** - Completable in 2-4 hour focused session
- **Value-focused** - Integrate technical enablers into value-delivering stories
